# 高效R编程-I/O指南

在临床研究中，我们经常需要处理数以万计的随访记录或庞大的多组学数据。很多指南都假设数据已经完美地存在于 R 中，但却忽视了将外部数据集导入 R 本身就是一个耗时且容易出错的过程。缓慢的数据读写和频发的格式报错，往往在项目起步阶段就消耗了研究者大量精力。

下面，俺将梳理 R 语言高效读写数据的策略，帮助大家更顺畅地对接临床数据库。

## 一、临床数据管理的核心铁律

{% hint style="warning" %}
在编写任何一行读取代码之前，必须牢记可重复科研的数据管理通用规则：**永远不要改写原始数据文件**。
{% endhint %}

* 原始文件应被严格设置为**只读状态**。
* 无论数据是从医院系统导出还是网络下载，请保留本地文件名不变，并在代码注释中注明其来源，这将极大提高项目的可复现性。

## 二、新手的万能钥匙：`rio` 包

对于不希望花费大量时间记忆各种数据读取函数的初学者，`rio` 包是名副其实的“多功能数据 I/O 包”。它能有效简化 R 中数据的导入与导出过程。

* **优势**：它只需要 `import()` 和 `export()` 两个极其直观的函数。
* **智能识别**：你无需指定数据格式参数，`rio` 会自动通过文件的后缀名（如 `.csv`, `.xlsx`, `.dta`, `.json` 等）推断并调用正确的底层读取包。

### 1. 极简导入数据 (`import`)

假设你有一份包含数千例肾脏肿瘤患者临床信息的 Excel 表格，只需一行代码即可将其加载为 R 的数据框，完全不需要去指定分隔符或特定的 Excel 读取参数：

```r
# 首先加载 rio 包
library("rio")

# 直接导入 Excel 格式的临床随访数据
tumor_data <- import("renal_tumor_patients_2024.xlsx")

# 导入逗号分隔的纯文本数据同样是这个函数
training_cohort <- import("RAPN_training_cohort.csv")
```

### 2. 极简导出数据 (`export`)

当你在 R 中完成了数据的清洗（例如修正了病理参数、剔除了缺失值），想要将整理好的队列输出分享给不使用 R 的临床同事时，只需改变文件名的后缀，`rio` 就会自动帮你转换格式。

```r
# 将清洗后的 R 数据框导出为 Excel 文件给同事审阅
export(tumor_data, "cleaned_renal_tumor_data.xlsx")

# 或者导出为 SPSS 格式供其他统计软件使用
export(training_cohort, "RAPN_cohort_for_analysis.sav")
```

## 三、纯文本文件（CSV/TSV）如何导入？

纯文本格式（如 `.csv`）因其简单性和跨平台兼容性，是临床数据分享的最常见格式。然而，R 语言处理文本文件有多种方式，选择不同的包将直接决定你的运算时间。

我们重点对比基础 R 的 `read.csv()`、`readr::read_csv()` 以及 `data.table::fread()`。

### 速度对比

对于超过 100MB 的大型队列数据集：

* `fread()` 与 `read_csv()` 几乎并驾齐驱，有望比基础的 `read.csv()` 快 5 倍左右。
* 即使是较小的数据集（1MB 左右），`fread()` 的速度也比基础包快约 5 倍。

### 速度背后的代价：稳健性(鲁棒性)考量

天下没有免费的午餐，`readr` 和 `data.table` 速度的提升一定程度上以牺牲极端的稳健性为代价。

* **类型推断策略**：与基础函数读取全表不同，快速函数仅基于前 1000 行样本数据来猜测列的数据类型。
* **异常值处理**：如果在第 1000 行之后突然出现文本异常值（例如某行年龄被误填为 "未知"），`readr` 会保留前 1000 行确定的数值型，并将异常记录强制转换为 `NA` 并抛出警告；而 `fread()` 则会自动将整列强制转换为字符型。
* **字符串处理**：对于明显的字符列，基础包默认转为因子（Factor），而快速函数会如实返回字符型。

| 函数                    | 输出结构              | 速度 | 对后排异常值的处理       | 字符串默认转化   |
| --------------------- | ----------------- | -- | --------------- | --------- |
| `read.csv()`          | `data.frame`      | 慢  | 整体扫描，类型稳定       | Factor    |
| `readr::read_csv()`   | `tbl_df` (Tibble) | 快  | 强制转为 `NA` 并警告   | Character |
| `data.table::fread()` | `data.table`      | 极快 | 整列降级为 Character | Character |

**应对超大文件**：如果一个 CSV 文件高达几 GB，直接读入会让普通电脑的内存崩溃。此时可以通过外部系统的命令行（如 Linux 的 `split -b100m`）将文件切分为多个 100MB 的易管理小块，再循环读入 R。

## 四、二进制存储：大幅缩减磁盘与等待时间

对于临床清洗完毕的数据，为了避免日后每次都重新跑一遍耗时的清洗脚本，推荐立即将其保存为二进制格式。二进制格式可以进行高效压缩，大幅缩小文件大小并成倍缩短读写时间。

### R 原生利器：`.Rds` 格式

这是 R 语言针对速度和压缩率高度优化的自有格式。

* **俺推荐使用 `saveRDS()` 和 `readRDS()`**。
* 很多教程通常使用 `save()` 和 `load()`，这非常容易出问题。因为 `load()` 会以原始变量名将其恢复到内存中，稍不小心就会覆盖掉你当前环境里同名的重要对象。而 `readRDS()` 非常灵活，强制你为其指定一个新的对象名，有效保障了工作空间的整洁。

### 跨语言协作桥梁：Feather 格式

如果你需要将整理好的数据交给团队的 Python 同事做进一步的分析，推荐使用 Feather 格式。

* 它具有轻量、速度极快且语言无关的特点。

在处理大规模数据时：

* **读入**：`read_feather()` 与 `readRDS()` 比普通的 CSV 快约 10 倍。
* **导出**：`write_feather()` 比保存 CSV 快 10 倍，而 `saveRDS()` 因为其强力的文件压缩机制消耗了计算资源，写入速度仅比 CSV 快 1.2 倍左右。不过，`.Rds` 文件体积通常远小于 CSV。

## 五、从外部网络直接拉取数据

随着公共数据库（如 GEO、TCGA）的普及，直接用代码拉取网络数据成为了必备技能。

* 使用 R 可以自动化网络下载与解压的过程，从而极大提高代码的可复用性。
* **最佳实践**：建议使用 `download.file()` 先将文件下载到本地硬盘（如 `extdata/` 目录），然后再用 `read_csv()` 读取。不要直接读取 URL 链接，这样可以避免因网络波动或报错导致反复漫长的重新下载过程。

```r
# 直接从 GitHub 等在线地址导入 JSON 或 CSV 格式的公共数据集，不要在正式脚本中这样做
library("rio")
online_reference <- import("https://example.com/raw/master/public_clinical_guidelines.json")

# 推荐下面这套工作流
library(readr)

# 定义数据的网络地址和本地保存路径
url <- "https://raw.githubusercontent.com/example-repo/clinical_data/master/large_renal_cohort_2024.csv"
dest_dir <- "extdata"
dest_file <- "extdata/large_renal_cohort_2024.csv"

# 首先检查本地是否已经有这个文件了，如果没有才执行下载
if (!file.exists(dest_file)) {
  
  # 如果连 extdata 文件夹都没有，就自动创建一个
  if (!dir.exists(dest_dir)) {
    dir.create(dest_dir)
  }
  
  message("检测到本地无该数据，正在从网络下载，请稍候...")
  # 使用 mode = "wb" (write binary) 可以确保在不同操作系统下下载文件不出错
  download.file(url, destfile = dest_file, mode = "wb")
  message("下载完成！")
  
} else {
  message("检测到本地已存在该文件，跳过下载步骤。")
}

# 3. 从本地快速读取数据
df_cohort <- read_csv(dest_file)
```
